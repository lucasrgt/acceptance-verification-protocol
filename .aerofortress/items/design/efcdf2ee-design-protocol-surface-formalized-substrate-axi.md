The design tier is now a first-class part of the PORTABLE AVP protocol (was implementation-only).

**Substrate axis** (`src/core/types.ts` → `type Substrate`): the layered-determinism axis — the
cheapest engine that can DECIDE a criterion. Values: `static` (host doctor), `dom` (React/DOM
adapter), `http` (backend adapter), `style` (jsdom computed style), `geometry` (headless
browser), `model` (LLM judge). Threaded through `core/dsl.ts` (`CriterionOptions.substrate` →
`CompiledCriterion` → emitted in `spec.criteria`, omitted when absent).

**Every design criterion declares its substrate**: the 7 jsdom archetypes `style`, the 7
geometry archetypes `geometry`, icon-correctness `model`. CLEANUP: the geometry archetypes had
overloaded `requires: 'geometry'` (a non-existent applicability seam — geometry subjects have
`render`/`url`, never a `geometry` seam, and design hooks don't implement `applies`, so it was
dead metadata) to MEAN substrate. Moved to the proper `substrate` field; `requires` now only
ever names a genuine seam (jsdom ones keep tokens/theme/…; icon keeps render).

**`protocol/design-catalog.json`** generated by `buildDesignCatalog()` (src/protocol.ts) from
`DESIGN_ARCHETYPES` (15) — shape `{protocol:'AVP', protocolVersion, catalog:'design',
substrates, oracleKinds, archetypes[]}`, each criterion carrying its substrate. Sibling to the
behaviour `protocol/catalog.json`, which stays BYTE-IDENTICAL (behaviour criteria have no
substrate → omitted). `SUBSTRATES` vocab + `DesignProtocolCatalog` type also in protocol.ts;
exported from `src/index.ts` (buildDesignCatalog, DESIGN_ARCHETYPES, SUBSTRATES, types).

**Drift guard** (`bench/protocol-sync.test.ts`) now covers BOTH catalogs via `it.each` —
regenerate both with `ASSAY_WRITE_PROTOCOL=1 npx vitest run protocol-sync`. Add a design
criterion without regenerating → red, same as behaviour.

**`docs/PROTOCOL.md`** documents the substrate axis, the design catalog, and conformance
("design substrates → a design adapter: style on jsdom, geometry on a browser, model on a
judge"); intro updated to "five substrates"; design-acceptance.md step 5 marked DONE.

Verified: tsc clean, full suite 51 files / 212 tests green (protocol-sync went 1→2 tests).
This closes the last STRUCTURAL item from the design-cycle; the only remaining design work is
Assay.NET (the .NET extraction will refine the Subject-descriptor + adapter contract).
Conceptually: AVP is ONE protocol, now with TWO archetype catalogs (behaviour + design) over
SIX substrates — "same protocol, sibling archetype catalog" exactly as the constituição frames it.
